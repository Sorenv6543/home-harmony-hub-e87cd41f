# New MCP tool

Claro exposes one authenticated MCP server at `/mcp` (`src/lib/mcp/admin.ts`, built with
`@lovable.dev/mcp-js`'s `defineMcp`). Every tool lives in `src/lib/mcp/tools/*.ts` and is built
with `defineTool`. All callers are authenticated via Supabase OAuth — there is no public/unauthenticated
MCP mount, so tools can assume a verified caller.

## Steps

1. **Pick a name and file**: kebab-case filename in `src/lib/mcp/tools/`, e.g. `list-properties.ts`.
   The tool's `name` field itself is snake_case (matches existing tools: `whoami`, `get_app_info`,
   `estimate_cleaning`).

2. **Write the tool** following this shape (see `src/lib/mcp/tools/estimate-cleaning.ts` or
   `whoami.ts` for full examples):

   ```ts
   import { defineTool } from "@lovable.dev/mcp-js";
   import { z } from "zod";

   export default defineTool({
     name: "tool_name",
     title: "Human-readable title",
     description: "What this returns and when to use it. State auth requirements explicitly.",
     inputSchema: {
       // Zod fields, or {} if no input. Use .describe() on each field.
     },
     annotations: { readOnlyHint: true, idempotentHint: true, openWorldHint: false },
     handler: async (args, ctx) => {
       if (!ctx.isAuthenticated()) {
         return { content: [{ type: "text", text: "Not authenticated" }], isError: true };
       }
       // ... logic ...
       const result = { /* structured data */ };
       return {
         content: [{ type: "text", text: "Human-readable summary." }],
         structuredContent: result,
       };
     },
   });
   ```

   - Set `annotations.readOnlyHint: false` if the tool mutates data; add `destructiveHint`/
     `idempotentHint` accordingly.
   - If the tool needs the service-role client, import it dynamically inside the handler:
     `const { supabaseAdmin } = await import("@/integrations/supabase/client.server");`
     — this file ships to the client bundle otherwise (see `src/integrations/supabase/client.server.ts`
     banner).

3. **Register it in `src/lib/mcp/admin.ts`** — this is the step that's easy to forget:
   - Add the import: `import myTool from "./tools/my-tool";`
   - Add it to the `tools: [...]` array in the `defineMcp({...})` call.
   - Optionally mention it in the `instructions` string so callers know it exists.

4. **Verify**: `bun run dev`, then check the tool is listed at `/mcp` (or via
   `src/routes/[.mcp]/list-tools.ts`, generated wiring — don't hand-edit that file).

## Gotchas

- `src/routes/mcp.ts`, `[.well-known]/oauth-protected-resource.ts`, and `[.mcp]/**` are
  generated by the `mcpPlugin` from `admin.ts` — never edit them directly; edit `admin.ts` and tool
  files instead.
- Don't add a public/unauthenticated tool — every caller goes through the OAuth issuer check.
